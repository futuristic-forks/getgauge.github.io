Why use Gauge?

  • Write business layer test specifications in rich Markdown format.
  • Write your test cases in Ruby, java... more to come
  • Execute via command line or Intellij
  • Build a living documentation.

How to install?

Download links and install information here

gauge -install

How to start?

1. Initialize a project
gauge --init java
gauge --init ruby 
	        
gauge --init java
 create  manifest.json
 create  specs
 create  specs/hello_world.spec
 create  env
 create  env/default
 create  env/default/default.properties
 create  src/test/java
 create  libs
 create  src/test/java/StepImplementation.java
 create  env/default/java.properties

Successfully initialized the project. Run specifications with "gauge specs/"

Choose the language the test cases are to written in. This initializes the project. A ruby project structure for example is: 

├── env
│   └── default
│       ├── default.properties
│       └── ruby.properties
├── manifest.json
├── specs
│   └── hello_world.spec
└── step_implementations
    └── step_implementation.rb
2. Executing Specifications
gauge specs/
gauge specs/hello_world.spec
